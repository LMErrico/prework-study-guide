HTML

  • "HTML attributes defines additional information about an element and are always defined in the beginning element."
  • "Semantic HTML elements indicate the role of their content, as well as to give structure to the content of the webpage."
  • "The head element contains information about the webpage."
  • "The body element represents the visible content shown to the user."
  • "The section elements define a section in a document that contains thematic content."
  • "An element refers to the entire element from opening tag to closing tag."
  • "A tag refers only to what appears inside the angle brackets."
  • "The element that holds all the child elements is known as the parent element."
  • "The elements that are nested inside other elements are called child elements."

CSS

  • Ways to style using CSS: inline CSS, internal CSS and external CSS.
  • "Separation of concerns is based on the idea that each section of code should have its own responsibility."
  • "Declarations contain the CSS property we want to apply and the value of the property."
  • "The margin property indicates how much space we want around the outside of an element."
  • "The padding property adds space around the content inside an element."
  • DRY coding principle: reuse or share pieces of code. i.e. use * and comma.
  • "When we create an external CSS file, we need to link it to the HTML file."
  • "Class attribute allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector."

Git

  • "Git is a version control system that lets you manage and keep track of each version of your proyect."
  • "To access Git, we use the command line."
  • "Git commands always start with git."
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: add a file as it looks now to your next commit.
  • giy pull origin main: pull to get the latest version.
  • "Git is version control software and GitHub is a cloud-based service that hosts our repos."

JavaScript

  • "JavaScript allows users to interact with websites."
  • "Variable is simply a named container that allows us to store data in our code."
  • "All strings are surrounded by quotation marks."
  • "Each line of JavaScript code ends with a semicolon."
  • "Control flow is the order in which a computer executes code in a script."
  • "JavaScript is read from top to bottom, and order counts."
  • "We can use conditional statements and loops to control the order in which code is executed, or if it is even executed at all."
  • "When creating an if statement, we provide one or more conditions for the computer to evaluate. If one condition is truthy, the computer will execute the code that we provide between curly brackets. But if the condition is not truthy, the code will not execute, interrupting the control flow. Interrupting the control flow in this way allows us to build apps that respond to a user's actions."
  • "An array is a single variable that is used to hold a group of data."
